        Now that you have installed all the required tools, you can finally install the Geode CLI!

        ### Scoop (recommended)

        If you have installed Scoop with the above steps, we recommend you use it to install Geode CLI, by entering the following command:

        ```bash
scoop bucket add extras
scoop install geode-sdk-cli
        ```

        You can also update the CLI later on by typing the following command:

        ```bash
scoop update geode-sdk-cli
        ```

        ### winget

        However, if you prefer to use a different package manager (like winget), you may do so by typing the following command:

        ```bash
winget install GeodeSDK.GeodeCLI
        ```

        You can later update the CLI by typing the following command:

        ```bash
winget upgrade GeodeSDK.GeodeCLI
        ```

        ### Manually (not recommended)

        Following instructions come from Geode docs:

        > Otherwise, you can manually install the Geode CLI by:
        > 1. Download the latest Windows release over on <Link href="https://github.com/geode-sdk/cli/releases/latest">GitHub</Link>
        > 2. Extract `geode.exe` into some folder on your computer
        > 3. Select the executable in File Explorer, <Kbd>Shift</Kbd> + <Kbd>Right-Click</Kbd> it and select `Copy as Path`
        > 4. Search `Edit the system environment variables` on Windows search. Alternatively, you can open up Control Panel and search for it, then select `Edit the system environment variables` or **to skip straight to step 6 select**
        > **`Edit environment variables for your account.`**
        > 5. Click `Environment Variables...`
        > 6. In the top `User variables` section, select the `Path` variable and click `Edit`
        > 7. Now click `New` and paste the path of the CLI executable you copied at Step 1. **Remove the `\geode.exe` from the end;** the path has to point to the *directory* with Geode CLI, not the CLI itself.
        > 8. Click OK to close the environment variable windows.

## Setting up Geode SDK

Now that you have installed Geode CLI, we may begin setting up Geode SDK!

<Steps>
    <Step>
    Before we install the SDK, we must create a profile so that the CLI knows where to install the compiled mods:

    ```bash
geode config setup
    ```

    Follow any instructions shown in the terminal from running this command.
    </Step>
    <Step>
    Install Geode SDK:

    ```bash
geode sdk install
    ```
    </Step>
    <Step>
    Install binaries for your current version:

    ```bash
geode sdk install-binaries
    ```
    </Step>
    <Step>
    (optional) You may switch to the nightly version by typing the following command:

    ```bash
geode sdk update nightly
    ```

    Or you may return to the stable verion by typing:

    ```bash
geode sdk update stable
    ```
    </Step>
</Steps>

## Setting up your IDE (optional, but recommended)

I recommend that you use VS Code as your IDE as it's cross-platform, lightweight, and Geode has a VS Code extension. To install VS Code, follow the steps below:

<Steps>
    <Step>
    Click the <Link href="https://code.visualstudio.com/Download">download link here</Link>.
    </Step>
    <Step>
    Open the installation wizard and go through all the steps shown on screen.
    </Step>
    <Step>
    Install the <Link href="https://marketplace.visualstudio.com/items?itemName=ms-vscode.cmake-tools">CMake Tools extension</Link>. This will allow you to easily build your mods with VS Code.
    </Step>
    <Step>
    Install the <Link href="https://marketplace.visualstudio.com/items?itemName=GeodeSDK.geode">Geode</Link> extension. This will allow you to easily look up sprites (and read the docs) and open GD straight from VS Code.
    </Step>
</Steps>

## Setting up your first mod

To create a new mod, run the following command:

```bash
geode new
```

The CLI will then prompt you several things:

```bash
| Info | This utility will walk you through setting up a new mod.
| Info | You can change any of the properties you set here later on by editing the generated mod.json file.
| Info | Choose a template for the mod to be created:
# We recommend you choose Default
Name: [Your Mod Name]
Version: [v1.0.0]
Developer: [Your Developer Name]
Description (optional): [Your description]
Location: [C:/ProjectsFolder/YourModPath]
| Okay | Do you want to add the cross-platform Github action? (Y/n) y
| Okay | Do you want to remove comments from the default template? (y/N) y
```

Next, open VS Code and do the following key combinations consecutively:

- <Kbd>Ctrl</Kbd> + <Kbd>K</Kbd>
- <Kbd>Ctrl</Kbd> + <Kbd>O</Kbd>

This will prompt you to select a folder. Choose your new mod folder.

<Callout variant="caution" icon="TriangleAlert">
If the CMake Tools extension isn't working for you, run the `geode build` command to build the mod.
</Callout>

Once your project is open in VS Code, open `CMakeLists.txt`. This will trigger the `CMake Tools` extension to activate. Once it does, press `F1` to open the command palette and type "Variant". You should see an option that says "`CMake: Select Variant`". Then, a new menu should appear, asking for the variant you want to build with. **Always select `RelWithDebInfo`**.

Finally, if you press `F1` again and type "Build", you should see an option that says "`CMake: Build`". This should compile the mod and automatically install it to your `gd/geode/mods` directory. Now, if you press `F1` and search "Launch", you should see an option that says "`Geode: Launch Geometry Dash`". If you click that, GD should open (make sure you have Steam open though).
